—   Date & Time with JavaScript   —

Date to String

String

The toString() method returns a string representing the specified Date object interpreted in the local timezone.

MDN: toString()
new Date().toString();

Date String

The toDateString() method returns the date portion of a Date object interpreted in the local timezone in English.

MDN: toDateString()
new Date().toDateString();

UTC String

The toUTCString() method converts a date to a string, interpreting it in the UTC time zone.

MDN: toUTCString()
new Date().toUTCString();

Time String

The toTimeString() method returns the time portion of a Date object interpreted in the local timezone in English.

MDN: toTimeString()
new Date().toTimeString();

ISO String

The toISOString() method returns a string in simplified extended ISO format,
which is always 24 or 27 characters long

YYYY-MM-DDTHH:mm:ss.sssZ

or

±YYYYYY-MM-DDTHH:mm:ss.sssZ,

respectively. The timezone is always zero UTC offset, as denoted by the suffix Z.

MDN: toISOString() Link: ISO 8601
new Date().toISOString();

ISO String   –   Slice, use only the first 16 characters

The slice() method extracts a section of a string and returns it as a new string, without modifying the original string.

MDN: slice()
new Date().toISOString().slice(0, 16);

ISO String   –   Slice, and replace the letter 'T' with a space, a dash, and a space ' - '

The replace() method returns a new string with one, some, or all matches of a pattern replaced by a replacement. The pattern can be a string or a RegExp, and the replacement can be a string or a function called for each match. If pattern is a string, only the first occurrence will be replaced. The original string is left unchanged.

MDN: replace()
new Date().toISOString().slice(0, 16).replace(/T/g, ' - ');

Methods to get elements of Date and Time

getFullYear()

The getFullYear() method returns the year of the specified date according to local time.

MDN: getFullYear()
new Date().getFullYear().toString();

getMinutes()

The getMinutes() method returns the minutes in the specified date according to local time.

MDN: getMinutes()
new Date().getMinutes().toString();

toLocal  –  Date, Time Methods

toLocaleString()

The toLocaleString() method returns a string with a language-sensitive representation of this date. In implementations with Intl.DateTimeFormat API support, this method simply calls Intl.DateTimeFormat.

MDN: toLocaleString()
new Date().toLocaleString();

toLocaleDateString()

The toLocaleDateString() method returns a string with a language-sensitive representation of the date portion of the specified date in the user agent's timezone. In implementations with Intl.DateTimeFormat API support, this method simply calls Intl.DateTimeFormat.

MDN: toLocaleDateString()
new Date().toLocaleDateString();

toLocaleTimeString()

The toLocaleTimeString() method returns a string with a language-sensitive representation of the time portion of the date. In implementations with Intl.DateTimeFormat API support, this method simply calls Intl.DateTimeFormat.

MDN: toLocaleTimeString()
new Date().toLocaleTimeString();

Date Time Format Constructor

Default

The Intl.DateTimeFormat() constructor creates Intl.DateTimeFormat objects that enable language-sensitive date and time formatting.

MDN: Intl.DateTimeFormat() constructor
new Intl.DateTimeFormat().format(new Date());

Date Style Full

The date formatting style to use when calling format(). Possible values include:

  • "full"
  • "long"
  • "medium"
  • "short"
MDN: Date Style Options
new Intl.DateTimeFormat('en-US', { dateStyle: 'full' }).format(new Date());

Time Style Full

The time formatting style to use when calling format(). Possible values include:

  • "full"
  • "long"
  • "medium"
  • "short"
MDN: Time Style Options
new Intl.DateTimeFormat('en-US', { timeStyle: 'full' }).format(new Date());

toLocale...String() Examples

Year

The representation of the year. Possible values are:

  • "numeric" (e.g., 2022)
  • "2-digit" (e.g., 22)
MDN: Year Option
new Date().toLocaleString(undefined, { year: 'numeric' });

Full Date

The representation of the weekday. Possible values are:

  • "long" (e.g., Thursday)
  • "short" (e.g., Thu)
  • "narrow" (e.g., T). Two weekdays may have the same narrow style for some locales
    (e.g. Tuesday's narrow style is also T)
MDN: Weekday Option 
new Date().toLocaleString(undefined, {
    weekday: 'long',
    year: 'numeric',
    month: 'long',
    day: 'numeric'
});

Full Date & Time   –   24-hour time, replace a comma ',' with a dash '-'

Whether to use 12-hour time (as opposed to 24-hour time). Possible values are true and false; the default is locale dependent. This option overrides the hc language tag and/or the hourCycle option in case both are present.

MDN: 12/24 Hour Time Option 
new Date().toLocaleString(undefined, {
    hour12: false,
    year: 'numeric',
    month: '2-digit',
    day: '2-digit',
    hour: '2-digit',
    minute: '2-digit'
  }
).replace(/,/, ' - ');

Time Zone Name

Time Zone Name   –   Date & Time

The localized representation of the time zone name. Possible values are:

  • "long" Long localized form (e.g., Pacific Standard Time, Nordamerikanische Westküsten-Normalzeit)
  • "short" Short localized form (e.g.: PST, GMT-8)
  • "shortOffset" Short localized GMT format (e.g., GMT-8)
  • "longOffset" Long localized GMT format (e.g., GMT-0800)
  • "shortGeneric" Short generic non-location format (e.g.: PT, Los Angeles Zeit)
  • "longGeneric" Long generic non-location format (e.g.: Pacific Time, Nordamerikanische Westküstenzeit)
MDN: time Zone Name Options
new Date().toLocaleString(undefined, {timeZoneName: 'long'});

Time Zone Name   –   Date

MDN: toLocaleDateString()
new Date().toLocaleDateString(undefined, {timeZoneName: 'long'});

Time Zone Name   –   Only

MDN: split()
